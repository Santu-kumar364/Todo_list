# Todo_list
### **Summary of Your Full-Stack Todo Application**  

Your **Todo App** is built using **React for the frontend** and **Spring Boot for the backend**, allowing users to **create, fetch, and delete tasks** seamlessly.  

---

### **Frontend (React) Overview**  

✅ **State Management:**  
- Uses `useState` to manage **user input (`title`)** and **list of tasks (`todos`)**.  

✅ **API Integration with Axios:**  
- Fetches, adds, and deletes todos from the backend.  
- Updates the UI dynamically without requiring a page refresh.  

✅ **UI Features (Styled with Tailwind CSS):**  
- **Task Input & Add Button:** Users can enter and add new tasks.  
- **Task List:** Displays todos dynamically.  
- **Delete Button:** Allows users to remove tasks instantly.  

---

### **Backend (Spring Boot) Overview**  

✅ **Controller-Service Architecture:**  
- Separates API logic (`TodoController`) from business logic (`TodoService`).  
- Uses a response model (`ApiResponse`) for consistent API responses.  

✅ **Key Functionalities:**  
- Stores and retrieves todos efficiently.  
- Handles user actions like adding and deleting tasks.
